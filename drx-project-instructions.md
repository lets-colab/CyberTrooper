### Supabase Edge Functions
**`notion-sync` (v6)** — Notion-first architecture:
1. Receives full form payload from browser
2. Writes to Notion FIRST (synchronously)
3. Inserts to Supabase with `notion_page_id` already attached, `sync_status: 'synced'`
4. Patches Notion with real `reference_id` and `supabase_id` after Supabase confirms
5. Fallback: if Notion fails → still saves to Supabase with `sync_status: 'error'`
6. Fallback: if Supabase fails after Notion succeeds → patches Notion with 'SUPABASE-SYNC-PENDING', returns success

**`notion-retry` (v1)** — Auto-recovery worker:
- Finds all rows where `sync_status IN ('error', 'new')` AND `notion_page_id IS NULL`
- Pushes each to Notion with full field mapping
- Updates `sync_status: 'synced'`, `notion_page_id`, `notion_page_url` on success
- Called by pg_cron every hour

## Bugs Fixed (Full History)

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| 401 Unauthorized on all form POSTs | `anon` role had RLS policies but no table-level GRANT | `GRANT INSERT, SELECT ON public.trooper_signups TO anon; GRANT USAGE ON SEQUENCE public.trooper_ref_seq TO anon;` |
| `payment_method: null` NOT NULL violation | `fd.get('payment_method') \|\| null` sent explicit null, overriding column DEFAULT | Changed to `\|\| undefined` + Object.fromEntries filter strips undefined values |
| Notion database empty (select mismatch) | Form sent "Under 500" but Notion expected "Under 1K" etc | Updated `<option value="">` to match Notion's exact select option names |
| Notion fire-and-forget (data loss) | Old arch called Supabase first, Notion was background fire-and-forget | Flipped to Notion-first synchronous; pg_cron retries failures hourly |
